# problem_solution — Pain → solution

> ID: `problem_solution`  
> Use when: establish **problem/cost**, then **resolution**. Often stacked with `product_ad`.

## When to use

- Pain, frustration, inefficiency, before/after, “tired of…”, “finally…”
- Problem + fix described (product optional until reveal)

## When not to use

- Feature carousel only → `product_ad`
- Pure mood → `mood_film`

## Logic

```
PAIN → (AGITATE) → SOLUTION → PROOF → RESOLVE
```

On 30s, merge AGITATE into PAIN if needed.

## Beat map — 30s

| Beat | Duration | Visual |
|------|----------|--------|
| `pain_hook` | 0–5s | Relatable pain — show, don’t lecture |
| `agitate` | 5–10s | Cost of pain (optional) |
| `solution_reveal` | 10–18s | Fix enters frame |
| `proof` | 18–25s | Better outcome (no fake stats) |
| `resolve` | 25–30s | New normal + optional CTA |

**Rule:** Solution should not appear before ~30% of runtime.

## With product_ad

Primary: `product_ad`. Secondary: `problem_solution`. Prepend `pain_hook`.

## Scene types

`pain_scene` · `solution_demo` · `relief_moment` · `result_state`

## Forbidden

- Fake metrics / testimonials
- Pain scenes user never mentioned
- Product in frame 1 with no setup

## Voiceover (~150 wpm)

30s: ~50–70 words. 60s: ~100–130 words.

## Example — ANC earbuds

| # | Beat | Shot |
|---|------|------|
| 1 | pain_hook | Loud subway |
| 2 | agitate | Missed focus / stress |
| 3 | solution_reveal | Earbuds on, world softens |
| 4 | proof | Calm work / call |
| 5 | resolve | Product lockup |
