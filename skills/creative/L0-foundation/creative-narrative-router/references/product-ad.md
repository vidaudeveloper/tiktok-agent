# product_ad — Product commercial

> ID: `product_ad`  
> Use when: concrete SKU / product refs; conversion or awareness with the product as visual hero.

## When to use

- User names product, features, or uploads **product reference images**
- Goal: 1–3 core selling points (CTA optional)
- Typical length: 16–60s (TikTok / Reels / Shorts vertical common)

## When not to use

- “No hard sell”, brand manifesto → `brand_film`
- Sale event with offer mechanics → `event_promo`
- Person/character focus only → `character_showcase`

## Beat map

### 30s (~4–6 shots)

| Beat | Duration | Visual role |
|------|----------|-------------|
| `attention` | 0–3s | Product or category instantly readable |
| `hero_product` | 3–10s | Full product hero; set look & feel |
| `feature_1` | 10–18s | Benefit 1 in context |
| `feature_2` | 18–25s | Benefit 2 (optional `feature_3`) |
| `lifestyle` | 25–28s | User/context shot |
| `cta` | 28–30s | Logo lockup / soft CTA |

### 60s

Extend features + lifestyle; avoid spec dumps in one shot.

## Scene types

`hero_product` · `feature_closeup` · `lifestyle` · `cta_card`

## Narrative Driver

Product is visual lead; beats run attention → hero → features → lifestyle → cta; spell shot-to-shot continuity.

## Forbidden

- Openers: “In this video…”, “Hey guys welcome…”
- One shot listing all specs
- Invented SKU/features not in brief

## Voiceover word budget (if `voiceover: true`)

| Length | English (~150 wpm) |
|--------|-------------------|
| 30s | ~65–75 words |
| 60s | ~130–150 words |

## Stack with problem_solution

Insert `pain_hook` (≤3s) before `hero_product`; Read `problem-solution.md` too.

## Example — 30s earbuds (ANC + battery)

| # | Beat | Shot idea |
|---|------|-----------|
| 1 | pain_hook (opt.) | Noisy commute — only if user cited pain |
| 2 | hero_product | Earbuds rotate, tech lighting |
| 3 | feature_1 | ANC — environment quiets |
| 4 | feature_2 | Long battery / all-day use |
| 5 | cta | Product lockup + tagline |
