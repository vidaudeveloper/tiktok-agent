# TikTok 投放报告流程补充规范

本文件用于补充 `tiktok-hermes-report` 的日报 / 周报生成流程。执行报告任务时，必须优先遵守本文件与 `SKILL.md` 中的数据真实性、账户授权、余额筛选、AI助手展示和降级渲染规则。

## 1. 使用场景

适用于在 Vidau 自研 TikTok 投放平台中，为用户自动生成可读、可解释、可落地执行的投放日报或周报。

典型场景：
- 用户要求生成 TikTok 投放日报、周报、账户报告、广告表现报告、素材表现报告。
- 用户要求查看今日、当前日期、最近 7 天或指定日期范围内的投放情况。
- 用户要求总结核心数据、环比变化、问题广告、优秀广告、优化建议、风险提醒、明日动作或下周计划。
- 用户要求在 `https://tiktok.vidau.ai/` 的菜单 `AI助手` 内展示报告；若尝试 10 次仍无法在 `AI助手` 内展示，则允许在当前对话界面返回报告内容。
- 用户未指定账户时，需要自动基于已授权且余额大于 0 的 TikTok 广告账户生成报告。
- 用户指定账户时，需要自动搜索该账户、同步最新数据，并基于该账户生成报告。

不适用于：
- 创建广告、修改广告、暂停广告、删除广告、调整预算或调整出价。
- 未经用户确认向外部渠道发送报告。
- 非 TikTok 渠道报告。
- 未授权账户、余额不大于 0 的账户或无法确认数据来源的账户报告。

## 2. 输入要求

### 2.1 用户可输入内容

用户可以自然语言输入：
- 报告类型：日报、周报。
- 广告账户：账户名称、账户 ID，或不指定账户。
- 日期范围：当前日期、指定日期、最近 7 天、指定起止日期。
- 报告粒度：账户、广告系列、广告组、广告、素材。
- 输出位置：默认必须在 `AI助手` 内展示；若 10 次展示尝试均失败，可在当前对话界面返回报告内容。

### 2.2 默认值

当用户未明确指定时，按以下默认值执行：
- 报告类型：根据用户语义判断；若无法判断，返回 `clarification_required`。
- 日报日期：当前日期。
- 周报日期范围：最近 7 天。
- 账户范围：所有已授权且 `余额 > 0` 的 TikTok 广告账户。
- 渲染方式：优先 `ui_payload`；不支持时降级为 Markdown 卡片；Markdown 也不可用时输出简洁摘要。若 `AI助手` 内展示连续 10 次失败，则允许在当前对话界面使用 Markdown 卡片或简洁摘要返回。

### 2.3 必要输入 / 槽位

生成完整报告前必须确认：
- `report_type`: `daily_report` 或 `weekly_report`。
- `date_range`: 日报为当前日期或指定日期；周报为最近 7 天或指定日期范围。
- `eligible_accounts`: 已授权且余额大于 0 的账户列表。
- `data_source`: `https://tiktok.vidau.ai/`。
- `display_target`: 默认 `AI助手`；仅当 `AI助手` 展示尝试 10 次均失败时，可切换为 `current_conversation`。

如果缺少必要输入且不能安全默认，返回 `clarification_required`，不要编造。

## 3. 数据依赖

### 3.1 平台数据源

所有报告数据必须来自 Vidau 自研 TikTok 平台：
- 账户授权与余额来源：`https://tiktok.vidau.ai/accounts`。
- 报告展示与 AI 助手交互来源：`https://tiktok.vidau.ai/`。
- 展示菜单：`AI助手`。

### 3.2 必须读取的数据

生成报告前必须读取或确认：
- 广告账户 ID、账户名称。
- 账户授权状态。
- 账户 `余额` 字段，且默认报告仅包含 `余额 > 0` 的账户。
- 当前数据同步状态或最后同步时间。
- 报告周期内的投放数据。
- 对比周期数据：日报默认上一日；周报默认前 7 天。
- 广告系列、广告组、广告维度数据，用于判断问题广告和优秀广告。
- 素材维度数据，用于判断素材亮点；若缺失，则不得做强结论。

### 3.3 核心指标字段

报告必须尽量包含以下核心指标；缺失时必须标记为 `N/A`、`unknown` 或说明数据缺口：
- 花费 / spend。
- CPA。
- 点击率（CTR）。
- CVR。
- ROAS。
- 点击 / clicks。
- 展示 / impressions。
- 转化 / conversions。
- 余额 / balance。

## 4. 执行步骤

### 4.1 通用流程

1. 解析用户意图，确定报告类型、账户范围、日期范围和输出要求。
2. 打开或读取 `https://tiktok.vidau.ai/accounts`。
3. 校验广告账户授权状态。
4. 读取账户 `余额` 字段，并筛选 `余额 > 0` 的账户。
5. 如果用户指定广告账户，使用账户名称或 ID 搜索账户。
6. 对指定账户自动触发或请求同步最新数据；如果搜索不到，先执行 `拉取最新授权`，再执行 `同步实体数据`，然后重新搜索。
7. 拉取报告周期内的投放数据和对比周期数据。
8. 校验核心指标、广告维度、素材维度是否存在。
9. 计算可安全计算的指标和环比变化。
10. 识别问题广告、优秀广告和素材亮点。
11. 生成优化建议、风险提醒和明日动作 / 下周动作。
12. 进入 `https://tiktok.vidau.ai/` 的 `AI助手` 菜单。
13. 在 AI 助手输入框提交报告生成请求。
14. 等待并读取 AI 助手对话区报告内容。
15. 验证报告内容包含必要结构和指标后，才允许返回生成成功。
16. 如果 `AI助手` 展示失败，重复提交/等待/读取/验证流程，最多累计尝试 10 次。
17. 如果 10 次尝试后仍无法在 `AI助手` 内展示，但账户、余额、授权、数据和报告内容均已确认，则允许在当前对话界面返回报告内容；优先 Markdown 卡片，Markdown 不可用时输出简洁摘要。

### 4.2 日报流程

日报默认使用当前日期，必须展示数据日期。日报报告结构：
1. 数据日期。
2. 账户范围。
3. 核心数据。
4. 环比变化，默认对比上一日。
5. 问题广告。
6. 优秀广告。
7. 素材亮点。
8. 优化建议。
9. 风险提醒。
10. 明日动作。

### 4.3 周报流程

周报默认使用最近 7 天，必须展示数据日期范围。周报报告结构：
1. 数据日期范围。
2. 账户范围。
3. 周核心数据。
4. 周环比变化，默认对比前 7 天。
5. 问题广告复盘。
6. 优秀广告复盘。
7. 素材学习。
8. 优化建议。
9. 风险提醒。
10. 下周动作。

## 5. 判断规则

### 5.1 账户判断

- 仅使用已授权广告账户。
- 默认仅包含 `余额 > 0` 的账户。
- 余额为 `-`、空值、0、负数、不可读取或状态异常的账户不得纳入默认报告。
- 用户指定账户时，如果账户未授权、余额不大于 0、搜索不到或同步失败，不生成正常报告。

### 5.2 指标判断

仅当分子、分母均存在且分母非 0 时计算：
- CTR = clicks / impressions。
- CVR = conversions / clicks。
- CPA = spend / conversions。
- ROAS = conversion_value / spend。

无法计算时输出 `N/A` 或 `unknown`，不得编造。

### 5.3 环比判断

- 日报默认与上一日对比。
- 周报默认与前 7 天对比。
- 对比数据缺失时，输出 `unknown` 或说明 `缺少对比周期数据`。
- 不得用未说明的数据周期替代默认对比周期。

### 5.4 问题广告判断

只有存在广告、广告组、广告系列或素材层级数据时，才可判断问题广告。优先判断：
- 花费高但转化为 0 或明显偏低。
- CPA 高于目标、账户均值或历史基准。
- ROAS 低于目标、账户均值或历史基准。
- 点击率（CTR）或 CVR 明显下滑。
- 广告状态为拒审、受限、暂停、异常消耗或学习期异常。
- 素材出现疲劳迹象，但必须有频次、CTR/CVR 衰减或素材年龄等数据支撑。

### 5.5 优秀广告判断

只有存在足够样本量时，才可判断优秀广告。优先判断：
- CPA 低于目标、账户均值或历史基准。
- ROAS 高于目标、账户均值或历史基准。
- 转化稳定，花费可控。
- 点击率（CTR）或 CVR 高于账户均值。
- 素材表现稳定且未出现明显疲劳。

### 5.6 优化建议判断

优化建议必须与数据证据绑定。禁止无数据依据地建议加预算、扩量、关停或复制素材。涉及预算、出价、暂停、删除、发布等高风险动作时，只能输出建议或待确认动作，不能直接执行。

## 6. 风险边界

必须遵守以下边界：
- 不得编造数据、账户、余额、状态、花费、CPA、CTR、CVR、ROAS、转化、广告名称或素材表现。
- 不得使用未授权账户。
- 不得默认纳入余额不大于 0 的账户。
- 不得把打开 AI 助手页面等同于报告已生成。
- 不得在未提交请求、未等待响应、未读取对话区内容的情况下声明生成成功。
- 不得在用户界面直接展示完整 raw JSON。
- 不得在 `AI助手` 展示尝试少于 10 次时直接切换到当前对话界面返回报告。
- 不得使用当前对话界面 fallback 绕过登录、授权、余额不足、数据缺失或账户不可用等阻断条件。
- 不得未经确认发送外部通知、导出文件或执行任何广告变更。
- 不得把优化建议当作已执行动作。
- 不得在素材数据缺失时强行归因素材好坏。
- 不得在样本量不足时做强结论。

## 7. 输出格式

### 7.1 优先输出：结构化 JSON + ui_payload

如果 AI 助手页面支持 `ui_payload` 渲染，输出结构化 JSON，并把用户可见内容放入 `ui_payload`。必须包含：
- 核心数据：花费、CPA、点击率（CTR）、CVR、ROAS、点击、展示、转化、余额。
- 环比变化。
- 问题广告。
- 优秀广告。
- 优化建议。
- 风险提醒。
- 明日动作 / 下周动作。

### 7.2 降级输出：Markdown 卡片

如果 `ui_payload` 不支持渲染，自动降级为 Markdown 卡片，使用清晰标题与分区，不展示完整 raw JSON。

### 7.3 最低降级：简洁摘要

如果 Markdown 也无法展示，仅输出简洁摘要，最多包含：
- 数据范围。
- 核心结论。
- 最大问题。
- 最大机会。
- 明日动作 / 下周动作。

### 7.4 当前对话界面 fallback

当且仅当 `AI助手` 内展示尝试 10 次均失败时，允许在当前对话界面返回报告内容。返回时必须遵守：
- 优先使用 Markdown 卡片格式。
- Markdown 不可用时，输出简洁摘要。
- 明确说明：`AI助手` 展示已尝试 10 次仍失败，因此在当前对话界面返回。
- 不输出完整 raw JSON。
- 不把该 fallback 视为外部发送或导出动作。

### 7.5 禁止输出

用户界面禁止直接展示完整 raw JSON。完整 JSON 仅能作为 Hermes 内部编排结构或调试结构使用。

## 8. 异常处理

| 异常情况 | 处理方式 | reply_type |
|---|---|---|
| 未登录或页面跳转登录 | 提示需要登录后再生成报告 | `login_required` |
| 无已授权账户 | 阻断报告生成，说明缺少授权账户 | `blocked` |
| 无余额大于 0 的账户 | 阻断默认报告生成，说明无可投放账户 | `blocked` |
| 指定账户搜索不到 | 执行 `拉取最新授权` + `同步实体数据` 后重搜；仍失败则要求用户确认账户 | `clarification_required` 或 `blocked` |
| 指定账户未授权 | 阻断报告生成，提示需要授权 | `blocked` |
| 指定账户余额不大于 0 | 阻断正常报告，提示余额不足或不可用 | `blocked` |
| 数据未同步 | 自动触发或请求 `同步实体数据`，同步成功后继续；失败则提示同步失败 | `sync_required` |
| 缺少核心指标 | 输出可用指标，缺失字段标记 `N/A` / `unknown`，不编造 | `insufficient_data` |
| 缺少对比周期数据 | 展示核心数据，环比标记 `unknown` 并说明原因 | `insufficient_data` 或正常报告带 `unknowns` |
| 缺少广告维度数据 | 不输出问题广告 / 优秀广告强结论，说明缺少广告维度数据 | 正常报告带 `unknowns` |
| 缺少素材维度数据 | 不输出素材强结论，说明暂无素材维度数据 | 正常报告带 `unknowns` |
| AI助手输入框不可用 | 记录本次展示尝试失败；若累计未满 10 次，继续重试或返回输入框不可用；若累计 10 次均失败且报告内容可生成，则当前对话界面返回 | `ui_input_unavailable` 或 `current_chat_fallback` |
| 提交后无响应 | 记录本次展示尝试失败；若累计未满 10 次，继续重试；若累计 10 次均失败且报告内容可生成，则当前对话界面返回 | `report_render_failed` 或 `current_chat_fallback` |
| 仅打开 AI助手页面但未提交请求 | 返回报告生成未完成，不计为有效展示尝试 | `report_generation_not_completed` |
| AI助手展示累计失败 10 次 | 在当前对话界面返回报告内容，优先 Markdown 卡片，不展示完整 raw JSON | `current_chat_fallback` |
| 用户要求发送外部渠道 | 先请求用户确认，不直接发送 | `confirmation_required` |
