# 广告创建详细流程

本文件为广告创建模块的详细参考流程，覆盖投放前检查、参数收集、创建流程、创建完成与异常处理。

## 触发关键词

创建广告、建广告、投广告、投放广告、我要投放、帮我投放、创建TikTok广告、搭建广告、批量创建广告、批量投放、帮我把广告发布出去、推广

## 投放前检查

识别到创建关键词后，在 VIDAU AGENT 端口进入参数收集流程。投放前须完成以下检查：

### 检查项
1. 账户授权状态 — 是否已授权、是否有效
2. 账户余额 — 余额是否充足
3. 推广目标 — 是否已选择（如网站访问、应用安装、转化等）
4. 优化目标 — 是否已选择（如转化、点击、曝光等）
5. 素材 — 是否已上传、是否审核通过
6. 定向 — 年龄、性别、地区、兴趣、行为、设备等是否配置
7. 预算 — 日预算/总预算是否填写
8. 出价 — 出价策略与出价金额是否配置（Maximum Delivery / Cost Cap）

### 检查不通过处理
若检查不通过：不允许进入创建提交，必须提示用户原因。

> 提示用户原因为：账户未授权 / 余额不足 / 推广目标未选 / 优化目标未选 / 素材未上传 / 定向未配置 / 预算未填写 / 出价未配置 等。

## 创建流程

按面板依次收集：广告账户 → 推广目标 → 优化目标 → 素材 → 定向 → 预算 → 出价。

### 预算检测规则
当用户选择同一个广告账户、同一个推广目标、同一个优化目标时，填写预算须检测 7 日内预算的平均值。

- 当前输入的预算超出或低于平均值的阈值范围时，提示建议预算范围。
- 提示格式：建议预算范围（基于近 7 日同账户/同目标/同优化目标的预算均值）。

## 创建完成

### 返回创建状态
创建完成后，必须返回创建状态：

| 状态 | 处理 |
|------|------|
| 成功 | 返回广告 ID、广告组 ID、Campaign ID 等创建结果；触发 AI 巡检询问 |
| 失败 | 触发失败重试规则 |
| 进行中 | 明确提示"创建进行中，等待接口返回"，不默认认为成功 |

### 失败重试规则
```
广告创建失败 → 自动重试最多 3 次 → 仍失败 → 返回失败原因 + 建议修改字段
```

### 触发广告 AI 巡检
创建成功后，触发广告 AI 巡检询问：
> "广告已创建完成，是否需要帮您创建广告巡检呢？覆盖花费、转化、CPA、ROAS、预算、素材等方面进行AI监测，及时掌握广告数据趋势，可以回复「确认」，进行开启"

用户回复"确认"或"开启"后，触发巡检模块（`references/ad-inspection-rules.md`）接管。

## 输出格式要求

广告创建在不同执行阶段，应根据当前执行状态返回对应的用户可见信息，并保证输出内容真实、可解释、可追踪。

1. 输出内容不得编造任何广告创建结果、TikTok 返回数据或广告状态
2. 若接口未返回结果，应明确提示当前状态，而非默认认为创建成功
3. 所有涉及广告创建、修改、发布等高风险操作，均需等待用户明确确认后方可执行

## 异常处理

### 1. 通用异常
所有异常均应优先返回明确原因，必须结合接口返回信息进行转换，不直接透传原始错误码。

### 2. MCP / TikTok 接口异常
- 自动重试次数为 3 次。
- 3 次仍失败，返回失败原因 + 建议修改字段。

### 3. 网络异常
- 若网络超时，重新请求一次。
- 若连续失败，提示网络异常。

### 4. 用户取消
若用户回复"取消""不用了""退出"：
- 立即停止创建流程。
- 返回"已取消广告创建"。
- 若已经成功创建且提交 TikTok，返回"本次创建已成功提交，无法终止"。
