# 日报/周报格式规范

本文件为日报/周报模块的详细参考规范，覆盖报表内容、输出格式、降级渲染策略与定时生成规则。

## 报表内容重点

### 日报重点
- 今日表现
- 环比变化
- 问题广告
- 优秀广告
- 素材亮点
- 风险提醒
- 明日动作

### 周报重点
- 本周趋势
- 问题归因
- 优秀广告复盘
- 素材可复制方向
- 下周动作

## 职责边界
- 自动检测是否有正在投放的广告（campaignsCount > 0）
- 自动检测是否有 7 天内花费大于 0 的广告账户/广告系列/广告组/广告
- 仅生成日报和周报

## 触发与日期/账户解析

### 触发关键词
日报、给我生成日报、给我生成周报、日报数据、昨天日报、今日数据报告、报告、周报数据、周报、本周周报、根据我的格式生成日报、根据我的格式生成周报

### 未指定日期和广告账户时
- 触发日报关键词：生成今日日报。自动回复："若没有指定日期和广告账户，将会为您生成今日日报"
- 触发周报关键词：生成本周周报（最多支持一个月内的周报数据）。自动回复："若没有指定日期和广告账户，将会为您生成本周周报（最多支持一个月内的周报数据）"

### 指定日期和广告账户时
支持的关键词格式：
- "x天日报"、"x日-x日的日报"
- "广告账户名称 xxx / 广告ID xxxx 的日报/周报"
- "广告系列名称 xxx / 广告系列 IDxxx 日报/周报"
- "广告名称 xxx / 广告IDxxx 的日报/周报"

根据指定的日期、对应的广告层级名称或输出的层级 ID 生成日报或周报。

## MCP 工具映射

| 功能 | MCP 工具 | 说明 |
|------|----------|------|
| 列广告主 | `list_advertisers` | 获取广告主列表、余额、campaignsCount |
| 日报指标 | `get_daily_metrics` | advertiser_id + start_date + end_date + level |
| 按 Campaign/Adgroup 细分 | `get_daily_metrics` (level=CAMPAIGN/ADGROUP) | 识别问题广告和优秀广告 |
| 同步数据 | `sync_advertiser_data` | ⚠️ 避免在交互查询中调用（极慢），手动在平台刷新 |

## 输出格式要求

### 数据指标展示
展示数据指标、环比变化、数据亮点。

环比变化约定（广告数据惯例）：
- 🔴 红色箭头向上 → 上涨
- 🟢 绿色箭头向下 → 下降

### 问题广告列表
以列表形式展示，字段如下：

| 字段 | 说明 |
|------|------|
| 日期 | 数据归属日期 |
| 广告名称 | 广告名称 |
| 广告ID | 广告 ID |
| 花费 | 该广告花费金额 |
| 点击率 | CTR |
| 转化率 | CVR |
| 曝光数 | 展示次数 |
| 存在问题 | 异常类型描述 |
| 优化建议 | 针对该问题的建议 |

### 优秀广告列表
以列表形式展示，字段如下：

| 字段 | 说明 |
|------|------|
| 广告名称 | 广告名称 |
| 广告ID | 广告 ID |
| 花费 | 该广告花费金额 |
| 点击率 | CTR |
| 转化率 | CVR |
| 曝光数 | 展示次数 |
| 建议 | 复制/扩量/保持建议 |

### 优化建议
- 分点列出
- 每条建议具体、可执行

### 明日关注/下周关注重点
- 列出需要关注的广告、指标、风险点

### 末尾询问
末尾询问需要帮用户执行哪些操作：
- 创建广告巡检
- 生成新的广告文案
- 创建新的素材
- 打开广告面板进行调整
- 再次同步数据
- 其他（根据用户回复识别操作）

## 渲染降级策略

按以下优先级降级渲染：

1. **支持 ui_payload 渲染** → 输出结构化 JSON + ui_payload（数据指标卡片、环比变化、问题广告列表、优秀广告列表）
2. **不支持 ui_payload 渲染** → 自动降级为 Markdown 卡片格式
3. **Markdown 也无法展示** → 仅输出简洁摘要，禁止在用户界面直接展示完整 raw JSON

## 格式要求

- 必须直接整理好的日报或周报形式，不要前言后语
- 分段清晰，标题明确，格式干净
- 字段缺失时，不要编造，留空或标注"暂无数据"

## 默认值与数据范围

- 日报：默认当前日期，展示数据日期
- 周报：默认 7 天内，展示数据日期
- 展示数据日期范围在报表头部明确标注

## 账户过滤规则

1. 默认日报/周报均仅针对已授权的广告账户进行生成。
2. 如果用户没有指定广告账户，则默认已授权的所有广告账户。
3. 需要过滤余额 > 0 的广告账户（`balance` 为 `null` 视为未同步，跳过）。
4. 若用户指定了广告账户，需要自动搜索该账户数据，自动同步最新数据。

## 定时生成规则

支持用户输入自定义的日报/周报格式，并支持定时生成发送日报/周报。

定时关键词示例：
- "每天x点给我生成日报"
- "每周星期x生成周报"
- "每周星期x下午n"

定时生成时：
- 使用 cronjob action='create' 创建定时任务
- 按用户指定时间自动触发对应报表生成
- 数据范围遵循上述默认值规则
- 输出格式遵循上述降级渲染策略
- 支持 cronjob action='remove' 取消定时

## 报告生成成功判定

只有满足以下全部条件，才算报告生成成功：
1. 已识别用户关键词进行操作
2. 已读取广告主列表
3. 已过滤余额 > 0 账户
4. 已获取日报/周报投放指标
5. 必须生成本地渲染或 Markdown 格式
6. 回复内容必须优先展示在页面上

若缺少数据，不要编造。
